# AI-Hub 597 원천 영상 filekey 선택 기준

작성일: 2026-05-29

이 문서는 AI-Hub `교통사고 영상 데이터` 데이터셋 597에서 원천 영상만 선별해 내려받기 위한 filekey 목록과 선택 기준이다. 전체 원천 데이터를 받으면 수십 GB에서 TB 단위까지 커질 수 있으므로, 영상 처리 검증에는 Validation 영상 중 작은 묶음부터 사용한다.

## 운영 원칙

- 원천 영상은 `.local/`, `datasets/aihub/.../source/`, 또는 `aihubshell` 다운로드 폴더 같은 로컬 전용 경로에만 둔다.
- 원천 영상 ZIP, 압축 해제 영상, AI-Hub API key는 Git, NAS 공유, 클라우드 업로드, 배포 산출물에 포함하지 않는다.
- 라벨 JSON은 평가 기준이고, 원천 영상은 OpenAI+YOLO가 실제 프레임에서 같은 정량 사실을 뽑는지 검증할 때만 사용한다.
- 기본 순서는 `SourceSmoke` → `SourceValidationRecommended` → 필요한 filekey 직접 지정이다.
- Training 원천 영상은 대부분 GB~TB 단위라 기본 사용하지 않는다. 꼭 필요할 때만 특정 filekey를 직접 지정한다.

## 선택 기준

- 사고대상 오염을 점검할 때: `SourceSmoke`를 먼저 사용한다.
- 보행자/자전거가 화면에 보여도 직접 충돌 대상이 아닌 경우를 더 확인할 때: 차대차와 차대자전거 Validation 영상 filekey를 추가한다.
- 신호등 오염을 점검할 때: `(신호등있음)` 파일을 우선한다.
- 도로 외 진입, 주차장, 회전교차로 같은 장소 오인을 점검할 때: `509464`, `509466`, `509467`을 우선한다.
- 모델이 특정 사고대상에서만 약하면 해당 축의 Validation 영상 filekey를 추가한다.
