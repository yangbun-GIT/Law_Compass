LawCompass

교통사고 AI 분석 도우미

영상, 사고 질문, KNIA 과실비율 인정기준, 법률 근거를 하나의 사용자 리포트로 연결하는 MSA 기반 AI 분석 시스템입니다.

Vue 3 Frontend Fastify Gateway FastAPI Agent Redis Stream Worker PostgreSQL + pgvector KNIA RAG
2026-05-30실제 코드 및 실행 가능한 로컬 시나리오 기준

Problem

사고 당사자가 바로 이해할 수 있는 결과가 필요합니다

사용자 입장

  • 영상과 사고 정황을 어떻게 설명해야 하는지 어렵습니다.
  • 과실비율 기준, 법규, 보험 절차가 분리되어 있습니다.
  • 전문가 문서를 그대로 보여주면 일반 사용자가 이해하기 어렵습니다.

시스템 목표

  • 사고 유형을 안정적으로 분류합니다.
  • KNIA 기준과 법률 근거를 함께 확인합니다.
  • 일반 사용자 화면과 고급진단 화면을 분리합니다.
  • 결과가 불확실하면 조건부 결과와 추가 확인 사항으로 표현합니다.

02Service boundaries

Runtime Flow

사용자 입력부터 리포트까지의 흐름

1. 사고 정보 수집

일반사용자모드 질문 흐름, 영상 업로드, 사고 유형 선택을 통해 분석에 필요한 사실을 구조화합니다.

2. 비동기 분석

Gateway가 작업을 생성하고 Worker가 Redis Stream을 통해 Agent 분석을 수행합니다.

3. 사용자 리포트

Agent와 Gateway가 KNIA 기준, 법률 근거, 과실비율, 행동 지침을 사용자 화면에 맞게 합성합니다.

03End-to-end flow

AI Agent

Agent는 “답변 생성기”보다 사고 판단 파이프라인에 가깝습니다

주요 단계

  • 프레임/영상 단서와 사용자의 답변을 병합합니다.
  • 사고 유형과 당사자 역할을 추론합니다.
  • KNIA 기준과 법률 근거를 검색합니다.
  • 과실비율, 조건부 결과, 추가 확인 질문을 구성합니다.

안전 장치

  • 영상 단서와 사용자 입력이 충돌하면 검토 필요 상태를 표시합니다.
  • 근거가 약한 경우 단정하지 않고 조건부 결과로 낮춥니다.
  • 일반 사용자 화면에서는 raw trace와 내부 진단값을 숨깁니다.

11User-facing separation

Gateway

Gateway는 인증, 라우팅, 리포트 합성을 담당합니다

Auth

세션 bootstrap, refresh retry, 보호 라우트 접근을 안정화하는 계층입니다.

Case API

케이스 생성, 영상 업로드, 분석 요청, 결과 조회를 프론트엔드에 제공합니다.

Report Composer

Agent 결과를 일반 사용자와 전문가 화면에 맞는 카드 단위 payload로 정리합니다.

12Gateway responsibilities

Worker

Worker는 분석 요청을 비동기로 처리합니다

Redis Stream 기반

Gateway가 분석 작업을 넣고 Worker가 가져가 Agent 호출, DB 업데이트, 상태 이벤트 발행을 수행합니다.

사용자 경험

프론트엔드는 분석 진행률과 단계 메시지를 표시하고, 완료 후 결과 화면으로 이동합니다.

13Async analysis

Storage

현재 기본은 로컬 저장소, 확장은 S3/NAS 어댑터입니다

로컬 개발

영상과 파생 파일은 로컬 storage 경로를 중심으로 처리됩니다. 발표 자료에는 실제 저장소 비밀값을 포함하지 않았습니다.

운영 확장

코드 구조상 S3/NAS SFTP adapter를 사용할 수 있지만, 이번 발표에서는 실제 endpoint, credential, secret을 노출하지 않습니다.

14Storage policy

Data

PostgreSQL과 pgvector가 분석 근거 저장소를 받칩니다

정형 데이터

사용자, 케이스, 파일, 분석 작업, 결과 payload는 관계형 테이블에서 관리됩니다.

검색 데이터

KNIA 및 법률 근거 검색에는 텍스트 매칭과 vector retrieval 기반 접근이 함께 사용됩니다.

17Security posture

Not MCP

현재 구조를 표준 MCP라고 표현하지 않습니다

LawCompass는 내부 API, Gateway, Agent, Worker, Redis Stream, PostgreSQL/pgvector를 조합한 서비스 구조입니다. 발표에서는 MCP라는 표준 프로토콜을 적용했다고 주장하지 않고, 실제 구현된 서비스 경계와 데이터 흐름만 설명합니다.

Internal API Agent orchestration Redis Stream RAG
